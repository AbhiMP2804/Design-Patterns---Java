We want to add additional functionalities to object at Runtime
we are delaing with the class exploion problem i.e due to permutation and combination there are lot of child classes are getting formed 
eg Mario game -> fly,heightup,gun,etc
We can infinitely stack up decorator 
Uses inheritance to behave like parent object(Is-A relationship) and changes its behavior using composition (Has-A) relationship 

Def - > Decorator pattern attaches additional responsibilities to an object dynamically.Decorator provides a flexible alternative to subclassing for extending functionality

Real World  Examples - Mario Game, Text Editors like google docs
